# Misho Kursova 🚀

**Misho Kursova** е перфектен Meme Generator (Генератор на мемета). Позволява добавяне на горен и долен текст към изображение по избор, както и запазването на творението в база данни.

Това хранилище изпълнява изискванията за: **Контейнеризация на проект с Docker Compose**.

## Структура на проекта

Проектът се състои от три основни компонента (контейнера):
1. **Frontend (React + Vite)**: Уеб потребителски интерфейс за създаване на мемета (въвеждане на текст и URL, превю на живо, Mystery Gallery). Комуникира с бекенда.
2. **Backend (FastAPI)**: REST API написано на Python. Отговаря за запазването/изтриването на създадените мемета.
3. **Database (MongoDB)**: NoSQL база данни за съхранение на записите с включена персистентност (Docker volumes).

```text
.
├── backend/
│   ├── Dockerfile
│   ├── main.py         # FastAPI приложение
│   └── requirements.txt
├── frontend/
│   ├── Dockerfile
│   ├── package.json
│   ├── src/            # React код
│   └── ...
├── compose.yml         # Docker Compose конфигурация
└── README.md
```

## Как работи всеки компонент и комуникацията между тях

- **Frontend-ът** работи на порт `3000` и визуализира интерфейса. Когато потребителят въведе URL на картинка и текстовете и натисне "Запази", фронтендът изпраща HTTP `POST` заявка (чрез Fetch API) към бекенда.
- **Backend-ът** работи на порт `8000`. Той е настроен с CORS, за да приема заявки от фронтенда. Когато получи заявката, използва асинхронния драйвър `motor`, за да запише данните.
- **Базата данни (MongoDB)** работи на вътрешен порт `27017` в Docker мрежата. Бекендът комуникира с нея чрез адреса `mongodb://database:27017` дефиниран в `compose.yml`. Използва се volume `mongo_data` за запазване на данните дори след спиране на контейнерите.

## Как се изграждат и стартират контейнерите

Проектът е изцяло контейнеризиран и изисква само инсталиран Docker и Docker Compose.

### Стартиране
Отворете терминала в основната директория на проекта и изпълнете:
```bash
docker compose up --build -d
```
Тази команда ще:
1. Свали MongoDB изображението от DockerHub.
2. Изгради (build) автоматично `frontend` и `backend` изображенията от съответните `Dockerfile` файлове.
3. Стартира всички контейнери в заден фон (`-d`).

### Достъп
След успешно стартиране, отворете браузъра си на:
- **Приложение (Frontend)**: [http://localhost:3000](http://localhost:3000)
- **API Документация (Backend)**: [http://localhost:8000/docs](http://localhost:8000/docs)

### Спиране
За да спрете проекта и контейнерите:
```bash
docker compose down
```
*(Забележка: Данните ви в базата данни ще се запазят, защото се използва Docker volume)*.
